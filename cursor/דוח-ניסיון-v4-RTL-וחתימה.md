# ניסיון v4 — עיצוב משופר + חתימה דיגיטלית

## מה השתנה

1. **תבנית:** python-docx (תוכן מלא ויציב) + עיבוד RTL ב-Word COM (מסגרת עמוד, כיוון קריאה)
2. **חתימה דיגיטלית:** תיקון קואורדינטות PDF — שדה `/Sig` ממוקם **מתחת לטקסט העוגן** בתוך המלבן התכלת
3. **מלבן חתימה:** הכיתוב מעל המלבן (כמו בדוגמה), לא בתוכו
4. **אימות:** בדיקה אוטומטית של סוג השדה + מיקום יחסית לעוגן

## תוצאות

| בדיקה | תוצאה |
|--------|--------|
| עמודים | **1** |
| שדה `MemberSignature` מסוג `/Sig` | **כן** |
| מיקום בתוך מלבן החתימה | **כן** |
| כל הבדיקות (`run_all_tests`) | **PASS** |

## קבצים לבדיקה

- PDF: `LetterGenerator/samples/fidelity_output/12345 כהן ישראל תחשיב זכויות אישי.pdf`
- תצוגה מלאה: `preview_page1.png`
- סימון שדה חתימה (מלבן אדום): `preview_signature_marked.png`

## אימות חתימה (אוטומטי)

```json
{
  "found": true,
  "field_name": "MemberSignature",
  "field_type": "/Sig",
  "is_signature_field": true,
  "rect": [135.66, 432.70, 315.66, 474.70]
}
```

מיקום: `ok: true` — השדה מתחת לטקסט "חתימה (שדה לחתימה דיגיטלית)" עם חפיפה אופקית.

## בדיקה ידנית (מומלץ)

1. פתח את ה-PDF ב-Adobe Reader / Foxit
2. **Fill & Sign** / מילוי וחתימה
3. ודא שניתן ללחוץ ולחתום **בתוך המלבן התכלת**
4. שמור את ה-PDF החתום ובדוק שהחתימה נשמרת

## הערות

- מסגרת העמוד וה-RTL מוחלים דרך Word לאחר בניית התבנית
- אם העיצוב עדיין לא תואם 100% לדוגמה — אפשר לספק את קובץ ה-`.docx` המקורי שלכם כבסיס
