## 1. מפת התיקייה — מה יש היום

```
מכונת מכתבים/
├── מסמך אפיון – מחולל מכתבים אוטומטי (.txt     ← שלך (מקור האמת)
├── תחשיב זכויות אישי/
│   └── תחשיב_זכויות_מעוצב.json                  ← שלך (טיוטה חלקית)
└── cursor/                                       ← שלי (דוחות, תכנון, הערות)
    └── דוח-תכנון-ופיתוח.md                      ← המסמך הזה
```

### 1.1 מה שלך (חומרי קלט)

| קובץ | תיאור | סטטוס |
|------|--------|--------|
| `מסמך אפיון – מחולל מכתבים אוטומטי (.txt` | אפיון מלא — ממשק, לוגיקה, טכנולוגיה, מבנה תיקיות | **מוכן** — מקור האמת לפיתוח |
| `תחשיב זכויות אישי/תחשיב_זכויות_מעוצב.json` | מיפוי עמודות Excel, תנאים, טקסטים | **טיוטה** — דורש התאמה (ראה סעיף 5) |

### 1.2 מה חסר אצלך (נדרש לפני הפקה מלאה)

| פריט | למה צריך |
|------|----------|
| **קובץ Excel לדוגמה** (xlsx/xls) עם כותרות עמודות אמיתיות | לאמת מיפוי עמודות, בדיקות תקינות, תצוגה מקדימה |
| **תבנית Word** (`תחשיב זכויות אישי.docx`) עם כל הטקסטים, הטבלה, והמשתנים | לפי האפיון — כל הנוסחים נערכים ב-Word בלבד |
| **מכתב מקורי לדוגמה** (PDF או Word) | לאימות שהפלט תואם למסמך המקורי |

## 2. מה אני אייצר (פלט הפיתוח)

כל הקוד, התשתית והקבצים הטכניים יישבו **מחוץ** לתיקיית `cursor/`, במבנה הפרויקט עצמו.

### 2.1 מבנה פרויקט מתוכנן

```
LetterGenerator/                    ← שורש הפרויקט (אייצר)
├── app.py                          ← נקודת כניסה
├── requirements.txt                ← תלויות Python
├── build.bat / build.spec          ← בניית app.exe (PyInstaller)
├── src/
│   ├── ui/                         ← מסך ראשי (PySide6)
│   ├── excel_reader.py             ← קריאת xlsx/xls
│   ├── template_engine.py          ← docxtpl + תנאים
│   ├── pdf_converter.py            ← המרה Word→PDF (LibreOffice/Word)
│   ├── signature_field.py          ← שדה חתימה אינטראקטיבי (pypdf)
│   └── letter_types/               ← לוגיקה לכל סוג מכתב
├── templates/                      ← תבניות (DOCX + JSON טכני)
│   ├── תחשיב זכויות אישי.docx      ← שלך (תעלה/תערוך)
│   └── תחשיב זכויות אישי.json      ← שלי (מיפוי טכני בלבד)
└── dist/
    └── app.exe                     ← גרסה מוכנה להפצה
```

### 2.2 מה נמצא בתיקיית `cursor/` (שלי בלבד)

| תוכן | תפקיד |
|------|--------|
| דוחות תכנון (כמו מסמך זה) | תיעוד החלטות, פערים, שלבי עבודה |
| רשימות משימות / סטטוס | מעקב מה בוצע ומה ממתין |
| הערות על קבצים שלך | ללא שינוי בקבצים המקוריים שלך |

**עקרון:** כל מה שב-`cursor/` = תיעוד ותכנון שלי. כל מה שמחוץ ל-`cursor/` = או שלך (אפיון, Excel, DOCX) או תוכנה שאני בונה.

## 3. איך אני מתכוון לבנות את התוכנה

### 3.1 טכנולוגיה (לפי האפיון)

| רכיב | ספרייה | תפקיד |
|------|--------|--------|
| ממשק | **PySide6** | חלון Windows פשוט — בחירת קבצים, ComboBox, Progress, לוג |
| Excel | **pandas + openpyxl** | קריאת עד 700 שורות, תמיכה ב-xlsx/xls |
| תבנית | **docxtpl** (Jinja2 ב-Word) | החלפת `{{משתנים}}` + בלוקים מותנים |
| PDF | **LibreOffice headless** או **Word COM** | המרה offline ל-PDF בפורמט A4 |
| חתימה | **pypdf** | הוספת שדה חתימה אינטראקטיבי ל-PDF |
| אריזה | **PyInstaller** | יצירת `app.exe` יחיד ללא התקנת Python |

### 3.2 ארכיטקטורה — זרימת עבודה

```
[Excel] ──קריאה──► [שורה + ערכים]
                         │
                         ▼
              [JSON: מיפוי עמודות + תנאים]
                         │
                         ▼
              [DOCX תבנית + docxtpl]
                         │
                         ▼
              [Word זמני] ──המרה──► [PDF]
                         │
                         ▼
              [pypdf: שדה חתימה]
                         │
                         ▼
              [שמירה: קוד חבר + שם משפחה + שם פרטי + שם מכתב.pdf]
```

### 3.3 מסך ראשי — התנהגות

1. **בחר קובץ Excel** — נתיב לקלט
2. **בחר תיקיית יעד** — לאן נשמרים ה-PDF
3. **סוג מכתב** — ComboBox (בשלב 1: רק "תחשיב זכויות אישי")
4. **שורה לתצוגה מקדימה** — מספר שורה (ברירת מחדל 1)
5. **תצוגה מקדימה** — מייצר PDF אחד ופותח אותו
6. **הפקת מכתבים** — לולאה על כל השורות, Progress Bar, לוג, דוח סיום

שגיאות (חשבון בנק חסר, סכום לא תקין וכו') — **לא עוצרות** את הריצה; נרשמות בחלון שגיאות וממשיכים לשורה הבאה.

### 3.4 הפרדה בין טקסט לטכני (לפי האפיון)

| ב-DOCX (שלך, ב-Word) | ב-JSON (שלי, טכני בלבד) |
|----------------------|--------------------------|
| כל הנוסחים, כותרות, פסקאות | שמות עמודות Excel לכל משתנה |
| טבלת התחשיב (עיצוב) | תנאי הצגת מקטעים (`L > 0` וכו') |
| כתב קבלה, הנחיות החזרה | שם קובץ פלט (תבנית שם) |
| משתנים: `{{FULL_NAME}}`, `{{P}}`… | מיפוי עמודות: `"P": "סהכ אחרי קיזוז"` |

> **חשוב:** לפי האפיון (שורות 359–361) — **לא** יישמרו נוסחי טקסט ב-JSON או בקוד.  
> הקובץ `תחשיב_זכויות_מעוצב.json` הקיים כולל מפתח `"texts"` — זה סותר את האפיון. בהמשך אעביר את הטקסטים ל-DOCX ואשאיר ב-JSON רק הגדרות טכניות.

### 3.5 תנאים לוגיים (מקטעים מותנים)

ב-DOCX, עם docxtpl:

```jinja2
{% if show_DEATH_SECTION %}
  ### תוספת בגין פטירה
  ...טקסט מהתבנית...
{% endif %}
```

המערכת תחשב מראש (לפי JSON) אילו דגלים `show_*` פעילים לכל שורה.

| מקטע | תנאי (לפי אפיון) | עמודה |
|------|-------------------|--------|
| תוספת פטירה | L > 0 | L |
| מענק עידוד עבודה | O > 0 | O |
| חבר במעמד קליטה | T = "נקלט.ת" | T |
| חוב בנייה פרטית | R אינו ריק | R |

### 3.6 שם קובץ הפלט

לפי האפיון:

```
{קוד חבר (עמודה C)} {שם משפחה (E)} {שם פרטי (F)} תחשיב זכויות אישי.pdf
```

דוגמה: `12345 כהן ישראל תחשיב זכויות אישי.pdf`

### 3.7 שדה חתימה ב-PDF

מלבן גדול בתחתית המסמך, מוגדר כ-**שדה חתימה אינטראקטיבי** (SigWidget) — ניתן לחתימה ב-Adobe Reader / Foxit, ללא אינטרנט.

## 5. פערים שזיהיתי — דורשים החלטה/קלט ממך

### 5.1 JSON קיים מול האפיון

| נושא | במצב הנוכחי | מה צריך |
|------|-------------|---------|
| מפתח `texts` | קיים ב-JSON | להעביר ל-DOCX; למחוק מ-JSON |
| `FULL_NAME` = F+E | ב-JSON | לאימות מול Excel — האם זה שם מלא נכון? |
| עמודות T, R | בתנאים אבל לא במיפוי | להוסיף ל-JSON |
| שם קובץ (C, E, F) | באפיון, לא ב-JSON | אוסיף `output_filename` ב-JSON |

### 5.2 המרה ל-PDF

שתי אפשרויות offline:

| אפשרות | יתרון | חיסרון |
|--------|--------|--------|
| **LibreOffice** (מומלץ) | חינמי, headless, עובד בלי Word | דורש התקנת LibreOffice על המחשב |
| **Microsoft Word** (COM) | עיצוב מדויק אם Word מותקן | תלוי ברישיון Word |

**שאלה אליך:** האם LibreOffice מותקן / מותר להתקין על המחשב שמריץ את התוכנה?

### 5.3 עיצוב הטבלה ב-DOCX

האפיון דורש טבלה "מודרני" — לא כמו Excel. העיצוב יוגדר **בתבנית Word** (שלך). אני אבנה את מבנה המשתנים; אתה תעצב ב-Word לפי הטעם.
